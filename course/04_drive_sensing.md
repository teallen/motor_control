# 04 驱动器与传感器工程

本章把视角从控制算法扩展到驱动器和反馈链路。机械臂规控算法看到的是关节位置、速度和力矩接口，但真实驱动器内部还要处理 PWM、母线电压、电流采样、编码器量化、速度估计、滤波和故障保护。

## 学习目标

学完本章后，你应该能判断：

- 编码器分辨率和采样周期如何影响速度估计。
- 低通滤波为什么能降噪，也为什么会引入延迟。
- 母线电压、电流采样和保护逻辑为什么会改变控制器的实际可用输出。
- 为什么驱动器带宽不是只由 PID 参数决定，还受采样、滤波、PWM 和保护限制。

## PWM、SVPWM 和母线电压

FOC 的电流环通常输出 `Vd/Vq`，经过逆 Park、逆 Clarke 得到三相电压指令，再通过 PWM 或 SVPWM 作用到三相逆变器。母线电压决定了可实现的电压矢量上限：

```text
电压需求过大 -> 电压矢量限幅 -> 电流环跟踪变差 -> 力矩输出不足
```

因此，高速下反电动势变大时，即使电流限幅还没到，电压限幅也可能先成为瓶颈。

## 编码器与速度估计

增量式或绝对式编码器给出的是离散位置。速度通常由差分得到：

```text
velocity = (position[k] - position[k-1]) / dt
```

这会放大量化误差和噪声。采样周期越小，差分对位置噪声越敏感；编码器分辨率越低，低速下速度估计越容易呈阶梯状。

本学习包提供：

- [include/motor_control/control/Filters.hpp](../include/motor_control/control/Filters.hpp)：一阶低通、速度估计、编码器量化。
- [examples/sim_encoder_velocity_filter.cpp](../examples/sim_encoder_velocity_filter.cpp)：编码器量化和速度滤波实验。

## 滤波的代价

低通滤波可以抑制高频噪声，但会引入相位延迟。对闭环控制来说，延迟会降低相位裕度：

```text
滤波越重 -> 噪声越小 -> 延迟越大 -> 可用闭环带宽越低
```

机械臂关节常见问题是：速度估计噪声大时，速度环和阻尼项会把噪声放大成力矩抖动；滤波太重时，又会让速度反馈变慢，导致响应发黏或振荡。

## 运行实验

```bash
cmake --build build
./build/examples/sim_encoder_velocity_filter
```

输出文件：

```text
outputs/encoder_velocity_filter.csv
```

建议画这些列：

- `true_position_rad` 和 `measured_position_rad`：看编码器量化误差。
- `true_velocity_rad_s`、`raw_velocity_rad_s`、`filtered_velocity_rad_s`：比较速度估计噪声和滤波延迟。
- `position_error_rad`：观察测量误差大小。

## 工程检查清单

- 编码器零点是否标定。
- 电机侧编码器和负载侧编码器是否存在减速器间隙或柔性差异。
- 速度估计是否在低速和静止附近抖动。
- 电流采样是否和 PWM 同步。
- 过流、过压、欠压、过温保护是否会触发控制降级或故障状态。

## 和机械臂规划的关系

规划器通常默认反馈是真实的 `q, dq`，但实际反馈经过量化、滤波和延迟。轨迹越激进，速度和加速度估计误差越容易进入控制回路，表现为：

- 轨迹拐点附近速度估计尖峰。
- 阻尼项把速度噪声转成力矩抖动。
- 接触控制中传感延迟导致力控发硬或振荡。
- 驱动器保护触发后，规划层仍以为底层可以继续跟踪。
