# 08 实时系统与安全状态机

电机控制和机器人关节控制是强实时问题。控制器参数本身稳定，不代表部署后稳定；周期抖动、通信延迟、时间不同步和故障处理都会改变闭环行为。

## 学习目标

学完本章后，你应该能判断：

- 控制周期和延迟为什么影响相位裕度。
- 电流环、速度环、位置环为什么通常运行在不同频率。
- EtherCAT、CAN/CANopen 这类通信为什么需要关注同步和超时。
- 安全状态机如何避免故障后继续输出危险力矩。

## 控制周期

典型伺服频率大致如下：

```text
电流环：10-40 kHz
速度环：1-5 kHz
位置环：0.5-2 kHz
上位机轨迹：100-1000 Hz
```

内环频率越高，越能及时控制电流和力矩；外环频率较低，但更依赖通信同步和轨迹插补。

## 延迟和抖动

固定延迟会带来相位滞后，周期抖动会让离散控制器实际 `dt` 波动：

```text
dt 变大 -> 控制积分和微分项变化 -> 等效闭环参数变化
```

真实系统中常见来源：

- 操作系统调度。
- 总线通信等待。
- 编码器和电流采样不同步。
- 日志、诊断或非实时线程抢占。

## 安全状态机

驱动器通常不会只有一个“运行/停止”开关，而是有明确状态：

```text
Boot -> Ready -> Enabled -> Fault
Fault -> Ready
```

本学习包的状态机在 [include/motor_control/system/SafetyStateMachine.hpp](../include/motor_control/system/SafetyStateMachine.hpp)。示例 [examples/sim_realtime_jitter_safety.cpp](../examples/sim_realtime_jitter_safety.cpp) 用周期抖动和过流条件触发 fault。

## 故障处理原则

- 检测到过流、过压、欠压、过温、编码器异常或看门狗超时，应立即进入 fault。
- fault 状态下不要继续输出使能力矩。
- fault clear 不应直接回到 enabled，而应回到 ready 等待重新使能。
- 上位机和驱动器都要能区分“命令停止”和“故障停止”。

## 和机械臂规划的关系

规划层如果不感知底层故障或延迟，会继续发送不可执行轨迹。工程上需要：

- 时间戳和周期检查。
- 轨迹缓冲区和插补。
- 关节限位、速度限位、力矩限位。
- 故障后轨迹取消或重新规划。
- 重新使能前的位置一致性检查。

这也是为什么机器人控制不只是算法公式，还包括实时系统和安全策略。
