## 推荐学习顺序

我建议按这个顺序补：

1. **先学电机模型和 PID**
   - 理解电压、电流、速度、力矩之间的关系。
   - 搞清楚为什么“电流近似等于力矩”。

2. **再学级联控制**
   - 最典型结构是：  
     `位置环 -> 速度环 -> 电流环 -> PWM -> 电机`
   - 做机械臂时，这个结构非常重要。

3. **重点学 FOC**
   - 现代机械臂关节常用 PMSM / BLDC。
   - FOC 是理解伺服驱动器的核心。
   - 至少要理解 `abc -> αβ -> dq`，以及为什么控制 `Iq` 就是在控力矩。

4. **补机器人关节控制**
   - 重点看力矩控制、阻抗控制、摩擦补偿、重力补偿。
   - 这些和机械臂规划、轨迹跟踪关系最密切。

5. **最后看驱动器和实时系统**
   - 比如 EtherCAT、CANopen、控制周期、通信延迟、故障保护。
   - 这些决定你的算法能不能稳定落地。

## 和机械臂规划最相关的重点

如果时间有限，优先学这些：

```text
力矩常数 Kt
电流环 / 速度环 / 位置环
FOC
关节力矩控制
逆动力学控制
摩擦补偿
阻抗控制
轨迹平滑性对控制的影响
驱动器限幅与饱和
控制周期与延迟
```

尤其是这条链要打通：

```text
规划轨迹 q(t), dq(t), ddq(t)
  -> 逆动力学算期望力矩 tau
  -> tau 转成电机电流 Iq
  -> FOC 控制三相电流
  -> 电机输出力矩
  -> 减速器传到关节
  -> 编码器反馈位置
```

## 推荐实践项目

最有效的实践路线：

1. 用仿真搭一个单电机模型，做 PID 位置控制。
2. 加入速度环、电流环，理解级联控制。
3. 用 Python / MATLAB / C++ 仿真一个 PMSM 的 FOC。
4. 做一个单关节模型，加入减速器、摩擦、负载惯量。
5. 实现重力补偿 + 轨迹跟踪。
6. 再扩展到 2-DOF 机械臂，看规划轨迹如何影响关节力矩和跟踪误差。

## 书和关键词

可以搜这些关键词：

- `Modern Control of Electric Drives`
- `Field Oriented Control PMSM`
- `Robot Joint Torque Control`
- `Impedance Control Robotics`
- `Computed Torque Control`
- `Servo Motor Control`
- `EtherCAT Servo Drive`
- `Motor Control MathWorks PMSM FOC`

如果只想先打基础，推荐路径是：

```text
PID 控制
-> 电机模型
-> 级联伺服控制
-> PMSM / BLDC
-> FOC
-> 机器人关节力矩控制
-> 阻抗控制
```

对你这种机械臂规划背景来说，最终目标不是成为电机驱动硬件专家，而是能理解“规划出来的轨迹为什么会让关节控制困难，以及驱动器实际能不能跟上”。这会直接提升你对轨迹平滑、动力学约束、力矩约束和控制可实现性的判断。
